# iOS 打包说明（自签名安装）

本项目在 Windows 上开发，本机无法直接编译 IPA（需要 macOS + Xcode）。以下两条路线任选其一。

---

## 路线 A：GitHub Actions 免费云端打包（推荐，无需苹果开发者账号）

产出**未签名 IPA**，下载后用爱思助手 / Sideloadly / AltStore 等工具以你自己的证书签名安装。

### 步骤

1. 在 GitHub 新建一个仓库（Public 免费不限时；Private 会消耗每月 macOS 构建额度，macOS 分钟按 10 倍计费）。
2. 在本项目目录执行（敏感数据已被 .gitignore 排除，不会上传）：

   ```cmd
   git add -A
   git commit -m "native ui"
   git remote add origin https://github.com/你的用户名/仓库名.git
   git push -u origin main
   ```

3. 打开 GitHub 仓库页面 → **Actions** → 左侧选 **Build iOS IPA (unsigned)** → 点 **Run workflow**。
4. 等待约 15-25 分钟，构建完成后在该次运行页面底部 **Artifacts** 下载 `ios-unsigned-ipa.zip`，解压得到 `app-unsigned.ipa`。
5. 用爱思助手「IPA 签名」或 Sideloadly，以你的 Apple ID / 证书签名后安装到 iPhone。

### 注意

- 免费 Apple ID 签名有效期 7 天，到期需重签；个人付费开发者账号签名为 1 年。
- 工作流文件：`.github/workflows/ios-ipa.yml`（macos-15 + Xcode 16，自动 prebuild → pod install → 免签名编译 → 打包 IPA）。

---

## 路线 B：EAS Build 官方云构建（需付费 Apple 开发者账号 $99/年）

EAS 直接用你的 Apple 证书签名，产出可直接安装的 IPA（Ad-Hoc 分发，设备需提前登记 UDID）。

```cmd
npm install -g eas-cli
eas login
eas build -p ios --profile adhoc
```

首次运行会引导你登录 Apple 账号并自动管理证书与描述文件。构建完成后控制台会给出 IPA 下载链接。
配置文件：`eas.json`（已内置 development / adhoc / production 三个档位）。
